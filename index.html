    Projects and Source Code

    1. HelloWorldPoject: The project that introduced the basic concepts: getting ADT going with Eclipse, setting up a project, creating and AVD and starting the emulator, intro to the Manifest.xml creating your first activity, starting a new activity at runtime via intents and passing parameter between activities.

    2. BarCodeExample: An example of how to use StartActivityForResult() to call an external application (ZXING) via a Intent to allow the user to scan and return back bar-code data.

    3. ContactPickerProject: Demonstrates how to use a targeted intent to launch and allow the user to pick a contact from the address book

    4. AllAboutListsProject: Code that demonstrates a plain-old listview, a listview that uses ListActivity and a custom Listview with performance optimizations.

    5. ServicesProject: A very simple project that demonstrates the concepts behind Android services. Please refer to the OpenExchangeProject for more advanced service example.

    6. AllAboutThreadsProject: Demonstrates the 2 supported threading mechanisms: Handlers and AsyncTask. Also demonstrates how to avoid activity leaks during when device configuration changes occur.

    7. BatteryBroadcastReciever: Example on how to use a BroadcastReceiver to listen for system events - this example demonstrates to monitor battery life in real time.

    8. LocationProject: Demonstrates how to work with location providers such as GPS and cell towers and get notified when the phone moves or when a user enters a proximity region you previously set.

    9. InternalContentProvidersProject: Demonstrates how to query the content providers exposed by Android. Examples includes querying the: phone call log, browser history and bookmarks, address book and phone settings.

    10. SensorTestProject: A test harness type application that would query your device for list of supported sensors and start reading data for each of them, dumping the raw output to screen.

    11. ShakeitBabyProject: Not written in class. Demonstrates the use of the accelerometer sensor to detect when the user shakes the device. Will come in handy if you want to build in a "Shake-to-Refresh" feature on your app.

    12. OpenExchangeProject: A (mostly) complete and (semi) useful app that was developed in stages. Uses almost all of the concepts learnt previously (Services, Alarms, WakeLocks, Network Access, Threading/ASyncTask, JSON parsing, Sqlite DB, BroadcastRecievers, SharedPreferences) plus a couple of extras (AutoComplete textfields and context menus). There will be a couple of more updates to this project over this weekend - will be adding a content provider to expose the data to other processes.

    Useful Links/Libraries/3rd Party Services

    1. Fake GPS: Allows you to fake your location in real time. Useful debugging tool when you are writing location based apps.

    2. RoboGuice: A useful Dependency Injection helper library for Android.

    3. Application Crash Report for Android (ACRA): Crash reporting made easy. Useful for QA and beta testing your app in the real world.

    4. Urban Airship: A cross-platform push notification server solution.

    5. Flurry: The best mobile cross platform analytics service out there (better than Google Analytics for Android, IMHO)

    6. SQLite Browser: A free tool to work with SQLite DBs (also see: SQLite Manager (Firefox extension)

    7. android-maven-plugin: An Android plugin for Maven build utility.

    8. Jackson JSON: The native JSON package shipped with Android is not very efficient when you need to parse complexly formatted or large chunks of JSON markup (uses CPU/battery and causes a lot of GC). For relatively simple markup such as the currency REST service we worked with, native JSON parser worked fine but if you want high performance, then Jackson is the JSON processor you should check out.

    9. Proguard: Is a tool that shrinks, optimizes and obfuscates your code by removing unused code and renaming classes, fields, and methods with semantically obscure names. The result is a smaller sized .apk file that is more difficult to reverse engineer.
